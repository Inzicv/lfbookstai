# Phase 3 — workflow Review Assistant

## Objectif

Aider à construire une review humoristique fidèle au style de l'utilisatrice sans remplacer sa créativité.

## Sources mémoire

- `memory/books/<slug>.md`
- `memory/reviews/reviews.md`
- `memory/humor/references.md`

## Étapes

### 1. Construction du pitch

Le Review Assistant propose :

- un résumé humoristique ;
- des analogies ;
- des références ;
- des métaphores ;
- des idées de blagues.

### 2. Construction de l'avis

L'utilisateur donne son avis brut.

Le Review Assistant :

- remet les idées dans l'ordre ;
- améliore la formulation ;
- conserve le ton habituel ;
- s'appuie sur les anciens `#avis`.

### 3. Propositions de hooks

Le Review Assistant propose 2 ou 3 hooks possibles.

### 4. Script final

Le Review Assistant assemble :

- hook validé ;
- pitch validé ;
- avis validé.

## Principes

Human In The Loop partout.

Le Review Assistant propose.

L'utilisateur décide.
